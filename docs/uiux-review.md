# UIUX Review

**レビュー日**: 2026-04-20  
**レビュー対象スナップショット**: フェーズ5完了時点（UI完成宣言段階）

---

## 1. 対象

- **プロダクト名**: FormCheck（フォーム確認に強い筋トレ記録PWA）
- **対象画面**: 全画面横断レビュー。特に重点的にレビューした画面は以下:
  - `/workouts/new` — セット記録入力（主対象）
  - `/dashboard` — ホーム
  - `/workouts` — 履歴一覧
  - `BottomNav` — 共通ナビゲーション
  - `/settings` — 設定
- **想定ユーザー**: 個人で筋トレを行う一般ユーザー。自分のフォームを見返したい人。ジムで片手にスマホを持ちながら使う。
- **この画面の目的**:
  - `/workouts/new`: トレーニング1セット分を素早く記録する。動画なしでも記録できる導線を提供する。
  - 全体: 「撮影→フォーム確認→記録→振り返り」のサイクルを心地よく回せるUIを実現する。

---

## 2. 総評

- **一言サマリ**: 骨格は良くできているが、**「ジムで片手で使う」リアルな利用シーンに対する詰めが甘い**。静かで上質な方向性は正しいが、心地よさの手前で止まっている。
- **このUIは心地よいか**: **まだ心地よくない**。余白と色の設計は方向として正しいが、フィールド高さのバラつき・ラベルの小ささ・操作導線の曖昧さが積み重なり、「なんとなく整ってはいるが、使っていて気持ちいいとは言えない」状態。
- **このUIは使われるか**: 現状では**2〜3回で離脱するリスクがある**。特にセット記録画面は、ジムでの連続入力に最適化されておらず、毎回のフルリセットがストレスになる。
- **大きな課題**: ①フォーム入力の効率が悪い（ジムでの連続記録に非対応）、②デザインガイドラインと実装の乖離が複数箇所で放置、③情報階層のメリハリ不足（静かすぎて、どこを見ればいいか迷う箇所がある）

## 3. デザイナーレビュー

### 良い点

- **カラートークンの一貫性**: `primary` / `surface` / `accent` / `muted` / `border` の5色体系がほぼ全画面で守られている。色数を絞る方針が効いていて、画面間で「別のアプリ」に感じることがない。
- **余白のリズム**: `(main)/layout.tsx` の `px-6 pt-12` が共通適用されており、ページ外側余白はガイドライン通り。画面を切り替えても余白が揃っている安心感がある。
- **アクセント色の抑制**: `bg-accent` はCTAボタンとダッシュボードの撮影カードに限定されており、「黄緑を使いすぎない」方針は概ね守られている。
- **タイポグラフィの意図**: `font-title` / `font-metric` / `font-caption` の使い分けに意図が感じられる。

### 気になる点

1. **フィールド高さの不統一**（ガイドライン違反）
   - `/workouts/new`: 種目select = `h-14`、RPE input = `h-12`、NumField = `h-14`
   - `/settings`: プロフィール入力 = `h-10`、保存ボタン = `h-10`
   - ガイドライン指定は入力フィールド `h-12`、ボタン `h-12`
   - **同一画面内ですら高さが揃っていない**のは、視覚的な不快感に直結する。

2. **角丸の不統一**
   - `/workouts/new`: `rounded-xl`（ガイドライン通り）
   - `/settings`: プロフィール入力が `rounded-lg`
   - 角丸のサイズは無意識に「別の系統」として認知されるため、統一すべき。

3. **BottomNavにアクセントバーがない**
   - ガイドラインでは「アクティブ: `text-primary` + ラベル下に細い `bg-accent` のバー（`h-0.5 w-5 rounded-full`）」と明記。
   - 実装は色の切り替えのみ。アクセントバーがないと、**アクティブ状態の認知が弱い**。特にグレー系のみの差分では「どこにいるか」が一瞬で伝わらない。

4. **ラベルが小さすぎる**
   - `/workouts/new` のフィールドラベル: `text-[10px] font-caption uppercase tracking-wider text-muted`
   - 10pxのウェイト300の `text-muted` は、明るい環境（ジム）で**ほぼ読めない**レベル。
   - ラベルは「静けさ」ではなく「機能」。読めないラベルは存在意義がない。

5. **select要素のスタイリング限界**
   - ネイティブ `<select>` はiOS/Androidで見た目のコントロールが効かない。カスタムセレクト or ボトムシートを検討すべき。
   - `h-14` でフォントが `text-base` なのは、他の入力（`text-sm`）と不揃い。

6. **textareaにラベルがない**
   - メモ欄はplaceholderのみでラベルなし。他のフィールドにはラベルがある。**一貫性が崩れている**。

7. **「既存の動画を紐付ける」リンクの視覚的弱さ**
   - `text-xs text-secondary` の小さなテキストリンク。フォーム下部に埋もれている。
   - 動画紐付けはこのアプリの差別化機能のはず。もっと存在感を与えるべき。

### 心地よさを損ねる点

- **フィールド高さが画面内でガタついている**。h-14, h-12, textareaの可変高さが並ぶと、フォーム全体が「整っていない」印象になる。
- **ラベルの極小フォント**が「上質」ではなく「読みにくい」になっている。静けさと可読性は両立させるべき。
- **保存完了画面から入力画面に戻るとき全フィールドがリセットされる**。同じ種目の連続セット記録で、毎回ドロップダウンから種目を選び直すのは苦痛。

### 改善提案

| # | 対象 | 提案 |
|---|------|------|
| D-1 | フィールド高さ | 入力フィールドは全て `h-12`、ボタンは `h-12` に統一。NumFieldの `h-14` は `h-12` に下げ、`text-lg` → `text-base` で調整。 |
| D-2 | 角丸 | 全フォーム要素を `rounded-xl` に統一。`/settings` の `rounded-lg` を修正。 |
| D-3 | BottomNav | アクティブラベル下に `<span className="mt-0.5 h-0.5 w-5 rounded-full bg-accent" />` を追加。 |
| D-4 | ラベル | `text-[10px]` → `text-[11px]`、`font-caption` → `font-label`、`text-muted` → `text-secondary` に引き上げ。 |
| D-5 | 種目select | カスタムセレクト（ボトムシート型）の実装を検討。MVPでは最低限 `text-sm` に統一。 |
| D-6 | textarea | `<Field label="メモ（任意）">` で包み、他フィールドと構造を揃える。 |
| D-7 | 動画紐付け | カード型UIに変更。`bg-surface rounded-xl px-4 py-3` で囲み、Videoアイコン付き。 |

## 4. ユーザーレビュー

### 良い点

- **「もう1セット記録」ボタンの存在**。保存後にすぐ次を記録できる意図は正しい。筋トレは連続でセットを記録する行為なので、この導線は不可欠。
- **数値フィールドのレイアウト**: 重量・回数・セットが横3列に並ぶのは、情報の種類として直感的にグルーピングされていて分かりやすい。
- **エラー表示**: `onBlur` でのインラインバリデーションは、入力途中で邪魔にならない。
- **動画撮影への導線カード**: 上部のカード型リンクは視覚的に目立ち、存在には気づける。
- **単位の表示**: 入力フィールド右端の「kg」「回」「set」は、何を入れればいいか迷わない。

### 気になる点

1. **ページを開いた瞬間の「何をすればいいか」が分かりにくい**
   - 最初に目に入るのは「動画を撮影して記録」カード。しかしこのページに来たユーザーは「動画なしで記録したい」はず（ダッシュボードの「さっと記録」から遷移）。
   - **最初に見せるべきものと、実際にやりたいことがズレている**。

2. **「または」の区切りが意味する心理的コスト**
   - 「動画を撮影して記録」→「または」→ フォーム、という構成は「本来は動画を撮るべきだが、それを省略している」という**罪悪感**を暗に生む。
   - 動画なし記録は「サブの手段」ではなく「正規の使い方」として扱うべき。

3. **「もう1セット記録」で全フィールドがリセットされる**
   - **これが最大のストレスポイント**。ベンチプレス3セットを記録するとき、毎回「ベンチプレス」をドロップダウンから選び直す必要がある。
   - 種目はそのまま保持し、重量・回数だけリセット（またはプリフィル）するべき。

4. **RPEが分からない**
   - 「RPE (任意)」とだけ書かれても、筋トレ初心者には何の指標か分からない。
   - ツールチップか説明テキストが欲しい。または「きつさ (1〜10)」のような平易な表現に。

5. **3列の数値フィールドが狭い**
   - iPhone SEサイズ（375px幅）で、3列 + 左右padding 24px = 各フィールド約100px幅。
   - 指が太いユーザーにとって、100px幅の入力欄は窮屈。特にジムで汗ばんだ手で操作する場面を想像すると不安。

6. **保存ボタンの位置が不安定**
   - フォーム内容の長さ（動画選択パネル展開など）によって保存ボタンの位置が変わる。
   - 画面下部に固定（sticky）するか、フォームの直後に十分なマージンを取るべき。

7. **保存完了画面のフィードバックが薄い**
   - チェックマーク + 「保存しました」はあるが、**何が保存されたのか** のサマリが小さい。
   - 保存直後の達成感が弱い。もう少し「やった！」感が欲しい。

### 心地よさを損ねる点

- **ジムでの連続記録フローが考慮されていない**。セット間の休憩中にサッと入力→保存→次のセット、のリズムが作れない。毎回の種目再選択が致命的。
- **「動画を撮影して記録」カードが毎回視界に入る**。動画なしで使うと決めたユーザーにとって、これは「お前はまだ動画を撮っていない」というプレッシャーになる。
- **ラベルの文字が小さすぎて読む気にならない**。ジムの照明下でスマホを見るとき、10pxの薄いグレーは認識できない。

### このままだと使われなくなる懸念

- **連続セット記録のたびに全リセット** → 3セット目で面倒になり、記録をやめる。
- **動画なし記録が「格落ち」に感じる構成** → 動画を撮らない日は記録自体をスキップするようになる。

### 改善提案

| # | 対象 | 提案 |
|---|------|------|
| U-1 | 種目の保持 | 「もう1セット記録」時は種目を保持。重量もプリフィル（前セットの値を初期値に）。 |
| U-2 | 動画カードの位置 | フォームの下部に移動、または折りたたみにして初期非表示。「セットを記録」が主役であることを明確に。 |
| U-3 | 「または」の削除 | 区切り線ごと削除。フォームを直接表示し、動画連携はフォーム末尾のオプションとして配置。 |
| U-4 | RPEの説明 | 「きつさ」に言い換えるか、`(i)` アイコンでツールチップ説明を追加。 |
| U-5 | 保存ボタンの固定 | `sticky bottom-20`（BottomNavの上）で画面下部に固定。 |
| U-6 | 完了フィードバック | 保存サマリのフォントを `text-base font-title` に拡大。簡易アニメーション（フェードイン）を追加。 |

## 5. PM/POレビュー

### 良い点

- **MVPとして必要十分な入力項目**: 種目・重量・回数・セット・RPE・メモ・動画紐付け。過不足なく揃っている。
- **撮影導線との接続**: 「動画を撮影して記録」カードでキャプチャ画面へ飛べる設計は、プロダクトの差別化ポイント（フォーム確認×記録）を体現している。
- **バリデーション設計**: 入力範囲チェックが適切（重量0-999、回数1-999、セット1-99、RPE 1-10）。
- **ダッシュボードからの動線分岐**: 「撮影して記録」と「さっと記録」の2パスはユーザーの状況に応じた分岐として正しい。

### 気になる点

1. **このアプリの最重要アクションが何か曖昧**
   - FormCheckは「フォーム確認に強い筋トレ記録PWA」。つまり**差別化は動画×フォーム確認**。
   - しかし `/workouts/new` 画面は「動画なし記録」が主体。**差別化機能が脇役に見える構成**になっている。
   - 撮影→記録のフロー（`/capture` → `/capture/meta`）と、記録→動画紐付けのフロー（`/workouts/new`）の関係が整理されていない。

2. **「既存の動画を紐付ける」が埋もれている**
   - 動画紐付けはこのプロダクトのコア機能の一つ。それが `text-xs text-secondary` の小さなリンクでは、**存在に気づかれないまま使われない**リスクが高い。
   - 「紐付けができること」をユーザーが知らなければ、このアプリはただの筋トレ記録アプリになる。

3. **記録の粒度問題: 「セット」か「ワークアウト」か**
   - 画面タイトルは「セットを記録」。1セットずつ記録する設計。
   - しかし一般的なトレーニングは「ベンチプレス 80kg × 8回 × 3セット」を**まとめて1エントリ**で記録したい場合もある。
   - 現状のUIでは「セット数」フィールドがあるのに「1セットずつ記録」という矛盾がある。セット数フィールドの意味が曖昧。

4. **日付の入力がない**
   - 機能要件（8-9）に「日付を登録できる」があるが、入力画面に日付フィールドがない。
   - 「今日の日付を自動で入れる」なら問題ないが、その旨の表示がない。過去の記録を追加入力できない。

5. **ホーム画面の情報密度**
   - ダッシュボードに「今週の記録」「直近の種目」「動画本数」のKPIカード、体重入力、グラフ、最近のセッション、動画一覧リンクが詰め込まれている。
   - MVPホームとしては情報過多。**ユーザーが最初にすべきアクション**がKPIに埋もれている。

### 心地よさを損ねる点

- **プロダクトの世界観がUIから伝わらない**。「フォーム確認に強い」がUIのどこからも感じられない。動画のサムネイル、フォーム比較のヒント、過去のフォーム改善の可視化など、**コアバリューを想起させる要素がゼロ**。
- **記録のセット数フィールドの意味が不明瞭**で、「これは何のための数字？」と迷う。

### このままだと使われなくなる懸念

- 動画紐付け機能の発見率が低く、**「ただの筋トレ記録アプリ」として認識**される → 競合（Strong, JEFIT等）との差別化を失う。
- ダッシュボードの情報量が中途半端で、**「見ても得るものがない」ホーム画面**になる → アプリを開く動機が薄れる。

### 改善提案

| # | 対象 | 提案 |
|---|------|------|
| P-1 | 動画紐付けの発見性 | フォーム下部にカード型UI（`bg-surface rounded-xl p-4`）で「動画と紐付ける」を常に表示。0件時は「撮影した動画をここに紐付けられます」の誘導テキスト。 |
| P-2 | 日付フィールド | 「今日」がデフォルトで選択された日付ピッカーを追加。過去日付も入力可能に。 |
| P-3 | セット数の整理 | 「セット数」フィールドを削除し、1セットずつの記録に統一するか、「3セット分まとめて保存」の挙動を明確にするか、方針を決定すべき。 |
| P-4 | ホームの情報整理 | KPIカードを削除または折りたたみ。「撮影して記録」「さっと記録」の2CTAを最上部に大きく配置。直近の記録は3件に絞る。 |
| P-5 | コアバリューの可視化 | 将来的に、記録に紐づいた動画のサムネイルを履歴一覧に表示。「フォーム確認に強い」を視覚的に体感させる。 |

## 6. 三者ディスカッションまとめ

### 一致した点

1. **連続セット記録時の種目リセットは最大のUX問題**（全員一致）
   - デザイナー: 「入力効率が悪い」
   - ユーザー: 「3セット目で面倒になって記録をやめる」
   - PM/PO: 「継続率に直結するため最優先で修正すべき」

2. **フィールド高さ・角丸の不統一は品質に見える**（全員一致）
   - 「静かで上質」を目指すなら、こういう細部のブレは許容できない。

3. **BottomNavのアクセントバー欠落はガイドライン違反**（全員一致）
   - 実装漏れであり、意図的な変更ではない。修正コストも低い。

4. **動画紐付け機能が埋もれている**（全員一致）
   - デザイナー: 「視覚的に弱すぎる」
   - ユーザー: 「気づかない」
   - PM/PO: 「差別化機能の発見率が低いのは致命的」

5. **ラベルのフォントサイズが小さすぎる**（全員一致）
   - ジムでの利用シーンを考えると、10pxのweight 300は読めない。

### 意見が割れた点

1. **「動画を撮影して記録」カードの位置**
   - デザイナー: 「撮影導線は重要。上部に残すが、カード面積を縮小」
   - ユーザー: 「動画なしで来たのに毎回見せられるのはストレス。下に移動」
   - PM/PO: 「撮影はコア機能。上部維持だが、初回以降は折りたたむか、ダッシュボードに任せて `/workouts/new` からは外す」
   - **結論**: `/workouts/new` からは削除し、代わりにフォーム下部で動画紐付けカードを目立たせる。撮影導線はダッシュボードとBottomNavに集約。

2. **セット数フィールドの扱い**
   - デザイナー: 「UIとしては3列で収まりが良い。残してよい」
   - ユーザー: 「セット数の意味が分からない。1セットずつ記録するなら不要」
   - PM/PO: 「まとめ記録のニーズもある。残すが、1セット記録時は省略可にする」
   - **結論**: Open Question。ユーザーテストで判断。現時点では残すが、placeholder を「1」にして省略時は1セットとみなす。

3. **保存ボタンのsticky化**
   - デザイナー: 「sticky化すると下部に常時黄緑バーが見え、画面が騒がしくなる」
   - ユーザー: 「スクロールして保存ボタンを探すより、固定の方が安心」
   - PM/PO: 「フォームが1スクロール以内に収まるなら不要。動画選択展開時のみ問題」
   - **結論**: 動画選択展開時のみstickyにするか、フォーム全体が1画面に収まるよう情報密度を調整する。

### 本質的な論点

- **「使っていて気持ちいい」に直結する改善は、連続記録フローの最適化**。これは見た目の問題ではなく**体験の問題**。
- **デザインガイドラインの意図（静かで上質）は正しいが、「静かすぎて機能が見つからない」領域に入っている箇所がある**。特に動画紐付け。
- **このアプリの本質的な課題は「フォーム確認に強い」がUI体験から伝わらないこと**。記録画面単体としては悪くないが、FormCheckならではの体験になっていない。

## 7. 最終合意

### Must（今すぐ直すべき）

| # | 内容 | 理由 |
|---|------|------|

## 8. 改善方針

### 削るもの

- `/workouts/new` 上部の「動画を撮影して記録」カードと「または」区切り → 画面の役割を「セット記録」に絞る
- ダッシュボードのKPIカード3つ → 「今週の記録」1つに集約するか、ホーム下部に移動
- ラベルの `uppercase tracking-wider` → 日本語ラベルに uppercase は無意味（英語の場合のみ有効）。tracking-wider も日本語では不要で、ラベルが横に広がるだけ

### 強めるもの

- **動画紐付けUI** → カード型に昇格し、0件時にも「紐付けられます」の誘導を常時表示
- **連続記録フロー** → 種目保持・重量プリフィルで「サッと入力→保存→次」のリズムを作る
- **保存完了のフィードバック** → サマリの文字サイズアップ、セット番号の表示（「ベンチプレス 3セット目を保存しました」）

### 静かにするもの

- **エラー表示** → 現在の `ring-2 ring-danger/40` は適度。これ以上派手にしない
- **動画選択パネル** → 展開時にページ全体が伸びすぎないよう、最大高さを制限してスクロール化
- **BottomNav** → 現在の控えめな設計は正しい。アクセントバーを足すだけで十分

### 気持ちよさを足すもの

- **保存時のマイクロインタラクション** → ボタンの `active:scale-[0.98]` に加え、保存成功時のチェックマーク出現アニメーション
- **数値フィールドの `+` / `-` ステッパー** → タップで0.5kgずつ増減。ジムでの片手操作に対応
- **種目選択後のフォーカス自動移動** → 種目選択 → 重量フィールドに自動フォーカス。入力リズムを作る
