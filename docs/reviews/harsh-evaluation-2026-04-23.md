## 3. ペルソナ別評価

### 3.1 トレーナー視点（4名）

#### トレーナーA（パーソナル歴8年・50代男性）

> 「これで何ができるの？ クライアントの動画を見て補助線を引いて…で、それを保存できないの？ 話にならない。」

- **致命的な問題**: 描画が保存されない（`console.log` のみ）。動画にコメントを残す機能もない。「次回はここに注意」というフィードバックが残らないならノートで十分
- **ワークアウト記録**: ブロック概念は理解できるが、トレーナーが4人の会員分を続けて入力する際、**テンプレート機能がない**のが致命的。毎回0から種目を選ぶのは業務に使えない
- **クライアント共有**: ない。Phase 3 予定とあるが、いつ来るか不明
- **月額評価**: **¥0（無料でも使わない）**

#### トレーナーB（ストレングスコーチ・40代女性）

> 「種目マスタが22種って少なすぎ。うちのジムの種目の半分もカバーできない。カスタム種目も追加できない。」

- `lib/mocks/movements.ts` に 22 種目がハードコードされている。ラットプルダウンはあるがケーブルローイング、レッグカール、カーフレイズ等がない
- カスタム種目の追加機能なし
- RPE（主観的運動強度）の入力欄がない（要件定義 `02-functional.md` にはあるのに未実装）
- セット間の休憩時間タイマーがない
- **月額評価**: **¥0**

#### トレーナーC（フリーランスPT・30代男性）

> 「動画のフォームチェックはアイデアとして面白いけど、グリッド線だけじゃ足りない。角度測定もできないし、骨格推定もない。これなら iPhone の標準カメラ＋マークアップで十分。」

- 補助線は 16分割グリッド・中心線の3種のみ。**カスタム補助線（任意位置への水平/垂直線ドラッグ）がない**
- 描画は線・矢印・円だけで、**角度測定・距離計測ができない**
- フレーム単位での進み/戻りは実装されているが、**2動画の並列比較はない**（Phase 2 予定）
- **月額評価**: **¥0**（「¥500/月ならβテスト協力するかも」）

#### トレーナーD（ジムオーナー兼トレーナー・40代男性）

> 「会員管理ができないし、複数ユーザーの切り替えもできない。ジムで使うなら最低限そこが必要。」

- マルチユーザー管理・ロール（トレーナー/会員）の概念がない
- ジム単位の管理ダッシュボードがない
- エクスポート機能がない（CSV、PDF等）
- **月額評価**: **¥0**

### 3.2 ジム初心者〜中級者視点（4名）

#### ユーザーE（ジム歴3ヶ月・20代女性）

> 「オンボーディングの3画面は可愛いけど、その後ダッシュボードに飛ばされて何していいか分からない。モックデータが入ってて、これ私のデータ？って混乱した。」

- **最大の混乱ポイント**: ログイン済みでもデータが空の場合、モックデータ（他人のワークアウト履歴）がそのまま表示される（`dashboard/page.tsx` L97-101）。自分のデータなのか分からない
- オンボーディング後の導線が弱い。「まず何をすべきか」が不明確
- 体重入力はできたが、翌日アプリを開いたときに**通知もリマインダーもない**ため忘れる

#### ユーザーF（ジム歴1年・30代男性）

> 「ワークアウト記録を作って保存したつもりが、次にアプリ開いたら消えてた。バグ？」

- **データ消失の恐怖**: ゲスト状態で保存ボタンを押すと `console.log` のみで実際には保存されない。しかもエラーメッセージや警告が**一切ない**（`workouts/edit/page.tsx` L116-118）
- ログイン状態でも保存失敗時はコンソールエラーのみ（L172-174）。ユーザーには何も伝わらない
- 前回の重量・回数が**自動入力されない**。毎回手入力は面倒

#### ユーザーG（ジム歴2年・20代男性）

> 「撮影機能は面白い。でもグリッド線を動画の上に重ねるだけなら、iPhone のマークアップと変わらない。補助線の位置を自分で調整できないのが惜しい。」

- 撮影〜動画保存の流れ自体は直感的（中央の撮影ボタン → 録画 → メタ入力 → 保存）
- ただし**撮影とワークアウトの紐付けが分かりにくい**。撮影後のメタ画面で「ワークアウトと紐付け」セクションがあるが、先にワークアウトを作っておかないと紐付け先がない
- 再生時の速度変更（0.25x/0.5x/1x）は良い
- **フレーム送り/戻し**は要件にあるが**未実装**

#### ユーザーH（ジム歴6ヶ月・30代女性）

> 「体重記録のグラフは見やすいけど、筋力の推移グラフがないのが残念。ベンチプレスの重量がどう伸びてるか見たい。」

- 体重・体脂肪率の推移グラフはある（`BodyDetail.tsx` のSVG手描き）
- しかし**種目別の重量・ボリューム推移グラフがない**。これがないと「成長してる実感」が得られない
- ワークアウト履歴画面は一覧のみで、**分析・傾向の表示がない**
- 「今日のおすすめ」はカテゴリ頻度ベースだが、**実際のトレーニング科学に基づいていない**（単にカテゴリ別に集計して頻度が低いものを提案）

### 3.4 デザイナー視点（1名）

#### デザイナーJ（UIデザイナー歴7年）

> 「ビジュアルのトンマナは統一されていて好印象。ただし、デザインシステムが『ドキュメントの中にだけ存在する』状態。実装との乖離が気になる。」

**良い点**:
- 「Quiet Premium Fitness」のトンマナが一貫している（モノクロ基調、`#3eed8d` のアクセント控えめ使用）
- Tonal を参考にした Fitness アプリらしい佇まい
- カード型UIの角丸 `18px` の統一、影の `0_0_0_1px_rgba(0,0,0,.04)` の統一は美しい
- BottomNav の中央撮影ボタンは Fitness アプリとして直感的

**問題点**:

1. **デザインシステムの形骸化**
   - `01-ui-design-guidelines.md` に詳細なガイドラインがあるが、**Figma/Storybook 等のビジュアルカタログがない**
   - コンポーネント仕様はドキュメントにあるが、実際のコンポーネントと1:1対応していないものがある
   - shadcn/ui 等の体系的UIライブラリを使っていないため、**同じパターンが各所で微妙に異なる実装**になっている（シートの背景色、角丸、padding が画面ごとに微妙にバラバラ）

2. **タイポグラフィの問題**
   - `text-[10px]`, `text-[11px]` が多用されている。WCAG では本文 `16px` 以上を推奨。特にジムの暗い照明下でスマホを見る利用シーンでは**読めない**
   - フォント Inter + Noto Sans JP の組み合わせは良いが、`font-metric` / `font-title` / `font-caption` / `font-label` のカスタム weight が 4種あり、開発者が使い分けに迷う

3. **色のアクセシビリティ**
   - `text-muted` / `text-secondary` のコントラスト比が WCAG AA（4.5:1）を満たしているか未検証
   - `bg-accent`（`#3eed8d`）上の文字色は `text-inverse`（白に近い？）→ 緑背景に白文字は**コントラスト不足の可能性大**

4. **空状態のデザイン不足**
   - データがない状態の画面が「モックデータ表示」か「空のリスト」のどちらかで、**適切な空状態イラスト・誘導がない**
   - ダッシュボードの「はじめよう」セクションは良いが、他の画面（動画一覧、ワークアウト履歴等）では空状態のデザインが弱い

5. **アニメーション・トランジション**
   - 画面遷移に `animate-slide-in` / `animate-slide-back` があるが、**CSS アニメーションの定義が見当たらない**（tailwind.config.ts にもない → 動いていない可能性）
   - シート表示のアニメーションが `transition-all` のみで、Fitness アプリとしてはチープ

| 観点 | スコア |
|------|--------|
| 迷わなさ | 6/10 |
| 継続しやすさ | 3/10 |
| 変化の実感 | 2/10 |
| 完成度 | 3/10 |

## 6. 技術的負債と構造上の問題

### 6.1 アーキテクチャ

| 問題 | 深刻度 | 詳細 |
|------|--------|------|
| 全画面 `"use client"` | 高 | Server Components を一切活用していない。初期表示が遅く、SEOも効かない |
| API Routes 0件 | 中 | クライアントから直接 Supabase。バリデーション・ビジネスロジックのサーバーサイド実行がない |
| `sessionStorage` でBlob URL受け渡し | 高 | タブ閉じ・ブラウザクラッシュでデータ消失。録画した動画が消える最悪のUX |
| グローバル状態管理なし | 中 | 画面間のデータ共有が URL パラメータと `sessionStorage` のみ |
| テスト 0件 | 高 | 品質担保の手段がない。リファクタリングが怖くてできない状態 |

### 8.3 一言まとめ

**「ガワは良い、中身がない。」**

UI のトンマナとデザイン方向性は Tonal を手本にした洗練されたものになっている。BottomNav の5タブ＋中央撮影ボタン、カード型UIの一貫性、オンボーディングの構成は評価できる。ドキュメントも充実しており、「何を作りたいか」のビジョンは明確。

しかし、**そのビジョンの 20% も実装されていない**。

- 保存できない記録アプリ
- 描画を残せないフォームチェックアプリ
- 前回の記録を見られない筋トレアプリ
- テンプレートのないトレーナー向けツール
- 通知もリマインダーもない継続支援

これらは「あったらいいな」ではなく、「ないと成立しない」レベルの欠陥。

**最優先アクション**: P0 の5項目（データ永続化・モック撤廃・エラー処理・CRUD・動画削除）を2-3週間で完了させ、「データが消えないアプリ」にすること。それができて初めてスタートラインに立てる。
