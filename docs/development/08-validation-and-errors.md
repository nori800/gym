## 1. 入力バリデーションルール

### 認証

| フィールド | 制約 | エラー文言（例） |
|-----------|------|------------------|
| メールアドレス | 必須、RFC 準拠形式 | 「有効なメールアドレスを入力してください」 |
| パスワード | 必須、8文字以上、英数混在推奨 | 「8文字以上で入力してください」 |
| パスワード確認 | 上と一致 | 「パスワードが一致しません」 |

### プロフィール

| フィールド | 制約 | エラー文言（例） |
|-----------|------|------------------|
| 表示名 | 必須、1〜30文字 | 「1〜30文字で入力してください」 |
| 身長 | 任意、50〜250 cm | 「50〜250の範囲で入力してください」 |
| 体重 | 任意、20〜300 kg | 「20〜300の範囲で入力してください」 |
| 目的 | 任意、最大200文字 | — |

### 動画アップロード／撮影

| フィールド | 制約 | エラー文言（例） |
|-----------|------|------------------|
| ファイルサイズ | ≤ 100 MB（ADR-003） | 「100MBを超えています。短い動画に分割してください」 |
| 動画長さ | ≤ 5分 | 「5分以内の動画を選択してください」 |
| 形式 | MP4(H.264) 推奨、ブラウザ再生可能形式 | 「この形式は対応していません。MP4をご利用ください」 |
| タイトル | 任意、最大50文字 | 「50文字以内で入力してください」 |
| メモ | 任意、最大500文字 | — |
| 種目タグ | 任意、リスト or カスタム（30文字以内） | — |

### トレーニング記録

| フィールド | 制約 | エラー文言（例） |
|-----------|------|------------------|
| 日付 | 必須、過去〜今日 | 「正しい日付を選択してください」 |
| 種目名 | 必須、1〜30文字 | 「種目名を入力してください」 |
| 重量 | 任意、0〜999 kg | 「0〜999の範囲で入力してください」 |

## 2. バリデーション実装方針

- **クライアント**（Zod スキーマ）で即時フィードバック。`lib/validations/` に定義する。
- **サーバー**（Route Handler / RLS）で最終チェック。クライアントを信用しない。
- フィールドごとに `onBlur` でバリデーション、送信時に全体チェック。
- エラーメッセージはフィールド直下にインラインで赤文字表示（`text-sm text-danger`）。

## 3. エラーハンドリングパターン

### 分類

| カテゴリ | 例 | UI 表示 |
|---------|------|---------|
| **入力エラー** | バリデーション失敗 | フィールドにインラインメッセージ |
| **通信エラー** | ネットワーク切断、タイムアウト | トースト「通信に失敗しました。再試行してください」＋再試行ボタン |
| **認証エラー** | トークン期限切れ、未認証 | ログイン画面へリダイレクト＋トースト「セッションが切れました」 |
| **権限エラー** | RLS 拒否 | トースト「この操作は許可されていません」 |
| **サーバーエラー** | 500系 | トースト「サーバーエラーが発生しました。しばらくしてから再試行してください」 |
| **カメラ権限拒否** | getUserMedia 失敗 | 画面内メッセージ「カメラへのアクセスが許可されていません」＋「ライブラリから選択」ボタンへ誘導 |
| **ファイル制約超過** | サイズ/長さ超え | アップロード前にブロック＋上記のインライン文言 |
| **ストレージ容量** | Supabase quota 超え | トースト「保存容量が上限に達しました」（MVPでは想定しにくいがハンドリングは入れる） |

### トーストの仕様

- 位置: 画面上部（`top-4 inset-x-4`）
- 自動消滅: 5秒。エラー時は手動閉じ可。
- 色: UI は [design/01-ui-design-guidelines.md](../design/01-ui-design-guidelines.md) に準拠。成功はダーク背景上で視認できる **エメラルド系**、エラー=`danger`、情報には **アクセント（イエロー）を濫用しない** — ニュートラルな `surface`＋`text-primary` でよい

### ローディング状態

| 操作 | 表示 |
|------|------|
| ページ遷移 | トップバーにスリムプログレスバー |
| フォーム送信 | ボタンを disabled ＋ スピナー |
| 動画アップロード | プログレスバー（%表示） |
| 一覧ロード | スケルトン（カード型のプレースホルダ） |

## 4. HTTP ステータスの扱い（Route Handler）

| ステータス | 意味 | フロント側処理 |
|-----------|------|----------------|
| 200 | 成功 | データ反映 |
| 201 | 作成成功 | 作成完了トースト＋遷移 |
| 400 | バリデーション失敗 | フィールドエラー表示 |
| 401 | 未認証 | ログインへリダイレクト |
| 403 | 権限なし | 権限エラートースト |
| 404 | リソースなし | 「見つかりません」画面 |
| 413 | ペイロード過大 | ファイルサイズ超過メッセージ |
| 500 | サーバー障害 | 汎用エラートースト |
